🔄 Behavioral Patterns

Patterns for how objects communicate and share responsibility: Strategy, Observer, Command, State, Chain of Responsibility, Template Method, Iterator.

Strategy

Define a family of interchangeable algorithms, encapsulate each one, and let the client pick which to use at runtime.

BehavioralGoFpayment processing

Observer

Define a one-to-many dependency so that when one object changes state, all its dependents are notified automatically.

BehavioralGoForder processing

Command

Turn a request into a standalone object, so it can be queued, logged, undone, or executed later.

BehavioralGoFdocument editor

State

Let an object change its behavior entirely when its internal state changes, as if it had switched to a different class.

BehavioralGoForder processing

Chain of Responsibility

Pass a request along a chain of handlers until one of them handles it, without the sender knowing which one will.

BehavioralGoFaccess control

Template Method

Define the skeleton of an algorithm in a base class, and let subclasses override specific steps without changing the overall structure.

BehavioralGoFreport generation

Iterator

Provide a way to access the elements of a collection sequentially without exposing how the collection is stored internally.
